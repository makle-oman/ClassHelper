# ClassHelper - 前端 UI 升级方案

## 1. 文档目标

本文档用于指导教师端前端页面的视觉优化与交互美化，目标是：

- 在不改动核心业务结构的前提下，提升页面完成度
- 统一首页之外的二级页面视觉语言
- 让静态页面更接近可交付产品，而不是开发中后台
- 为后续让 Claude 或其他协作模型直接执行 UI 优化提供明确规范

本文档默认前提：

- 保留现有绿色主品牌色方向
- 保留现有页面路由和主要功能结构
- 不追求“推倒重做”，以渐进优化为主
- 优先优化教师端高频页面

---

## 2. 当前 UI 状态判断

### 2.1 当前最成熟的页面

- 首页 `app/(tabs)/index`
- 我的 `app/(tabs)/profile`

这两页已经具备较强的产品感，主要优势如下：

- 顶部有明确的视觉焦点
- 区块划分清晰
- 主色使用统一
- 具有首页/个人中心应有的“封面感”
- 信息展示与操作入口平衡较好

### 2.2 当前主要问题

除首页和“我的”之外，其他二级页面的主要问题不是“难看”，而是“过于工具页”：

- 顶部通常只有一个普通导航栏
- 进入页面后直接是筛选或列表，缺少页面身份感
- 搜索栏、筛选 Chip、弹窗按钮、浮动按钮虽然都可用，但规格还不够统一
- 统计信息通常散落在页面中，层次不够集中
- 有些页面已经有不错的内容，但视觉重心不够明确

换句话说，当前问题是“完成了功能原型”，但还没有完全达到“成熟产品页面”的视觉水准。

---

## 3. 设计方向

### 3.1 整体方向

整体风格建议继续保持：

- 温和、清爽、可信赖
- 偏教育工具，不做炫技风
- 信息清楚，操作明确，视觉不过载

关键词：

- 清新
- 稳定
- 亲和
- 秩序感
- 轻量质感

### 3.2 不建议的方向

以下方向不建议采用：

- 过深的暗色强对比风格
- 过多渐变与重阴影堆砌
- 每个页面都做大视觉头图，导致风格失控
- 大量高饱和标签色同时出现
- 把所有业务页都改成首页式大卡片封面

建议做法是：

- 保持首页和“我的”作为视觉标杆
- 让高频业务页有轻量头部和摘要感
- 让管理页更有结构和层次
- 让通用控件统一起来

---

## 4. 全局 UI 规范

## 4.1 页面基础结构

教师端主要页面建议统一成以下结构：

1. 顶部导航区
2. 页面摘要区
3. 操作区
4. 筛选/统计区
5. 主内容区

并不是所有页面都必须拥有这 5 段，但高频页面至少应拥有：

- 顶部导航区
- 页面摘要区
- 主内容区

### 推荐结构模板

```text
顶部导航
页面摘要
操作区（搜索 / 新增 / 导入 / 切换）
统计或筛选
列表 / 图表 / 表格主内容
```

---

## 4.2 布局与间距

### 页面边距

- 页面左右内边距统一：`20`
- 小模块左右内边距可用：`12` 或 `16`
- 顶部内容区与导航栏间距统一：`10-16`

### 区块间距

- 小区块之间：`8`
- 同级模块之间：`12`
- 大区块之间：`16-20`

### 卡片内边距

- 普通信息卡：`14-16`
- 大摘要卡：`18-20`
- 弹窗内容区：`20`

建议项目内尽量只保留三档间距：

- `8`
- `12`
- `16/20`

避免每页出现大量 `7 / 9 / 13 / 18` 这种零散数值。

---

## 4.3 圆角与阴影

### 圆角

- 搜索栏、输入框、按钮：`10-12`
- 普通卡片：`14-16`
- 大摘要卡、弹窗：`18-20`
- 圆形状态点、头像：完整圆角

### 阴影

建议统一使用轻阴影，避免不同页面阴影差异过大：

- 阴影透明度偏低
- 以“浮起一点点”为目标
- 不要用特别重的 elevation

优先表达方式：

- 用留白和层级表达卡片，而不是强阴影

---

## 4.4 字体层级

### 页面级

- 页面主标题：`17-18 / 700`
- 页面副标题或说明：`12-13 / 400-500`

### 分区级

- 分区标题：`15-16 / 700`
- 分区辅助说明：`11-12 / 400`

### 内容级

- 卡片主标题：`14-16 / 600`
- 卡片次信息：`12-13 / 400-500`
- 弱提示：`10-11 / 400`

### 数据级

- 重要统计数字：`18-24 / 800`
- 小统计数字：`14-16 / 700`

建议统一信息优先级：

- 主标题最突出
- 次标题和业务标签稍弱
- 日期、备注、说明最弱

---

## 4.5 颜色规则

### 主色

- 继续使用绿色主品牌色
- 绿色主色主要用于：
  - 主要按钮
  - 激活态
  - 页面重点数字
  - 核心强调信息

### 状态色

建议所有页面统一使用以下语义色：

- 成功：绿色
- 信息：蓝色
- 警告：橙色
- 危险：红色

### 使用原则

- 状态标签优先使用“浅底 + 深字”
- 不要每个模块单独发明一套颜色体系
- 同一含义在不同页面尽量同色

例如：

- “已完成 / 已通过 / 正常”统一偏绿色
- “待处理 / 待录入 / 临近截止”统一偏橙色
- “异常 / 拒绝 / 缺席 / 紧急”统一偏红色

---

## 5. 通用控件优化规范

## 5.1 顶部导航栏

### 当前问题

- 大部分业务页各自手写导航栏
- 标题、返回区、右侧操作位写法接近但还没完全统一
- 空右侧占位和有右侧按钮时，视觉不完全稳定

### 统一结构

建议统一为：

- 左侧固定返回按钮区域
- 中间标题区域
- 右侧固定操作区域或等宽占位

### 规范建议

- 左侧返回区宽度固定
- 右侧操作区宽度固定
- 返回图标大小统一 `20-24`
- 点击热区统一 `40x40`
- 标题始终视觉居中
- 导航栏底部线条、内边距统一

### 不建议

- 标题跟着左右内容偏移
- 有些页面右侧空白，有些页面没有占位
- 返回按钮可点区域过小

---

## 5.2 搜索栏

### 统一规范

- 高度统一：`44`
- 圆角统一：`12`
- 左侧图标统一
- 输入后右侧显示清空按钮
- 边框颜色和聚焦态统一

### 建议状态

- 默认
- 聚焦
- 已输入
- 空结果

### 视觉建议

- 搜索栏和导入/新增按钮应视作同一个操作模块
- 搜索栏本体不要太高，也不要做得太厚重

---

## 5.3 筛选 Chip

### 建议分两类

- 筛选 Chip：用于切换班级、类型、标签
- 状态 Chip：用于展示状态，不一定可点击

### 统一规范

- 高度统一：`32-34`
- 内边距统一
- 未选中：浅灰底或白底描边
- 选中：主色浅底 + 主色描边 + 主色文字
- 特殊状态可用对应语义色浅底

### 不建议

- 某些页面只改文字颜色，某些页面只改背景
- 有些 Chip 像按钮，有些 Chip 像标签，缺乏一致性

---

## 5.4 弹窗底部按钮

### 统一结构

- 左侧次按钮：取消 / 返回 / 关闭
- 右侧主按钮：确认 / 保存 / 创建 / 提交

### 统一规范

- 按钮高度统一：`44`
- 圆角统一：`12`
- 字重统一：`600-700`
- 左右按钮间距统一：`10`
- 底部区域留白统一

### 文案建议

尽量统一为以下形式：

- 取消 / 确认
- 取消 / 保存
- 取消 / 创建

危险操作再单独使用红色，例如：

- 取消 / 删除

---

## 5.5 浮动操作按钮 FAB

### 统一分类

- 主 FAB：实心主色，优先级最高
- 次 FAB：白底或浅底描边，优先级次之

### 统一规范

- 统一放置于右下角
- 与底部和右侧边距统一
- 高度统一
- 圆角统一
- 阴影统一

### 多按钮时的规则

- 主按钮更醒目
- 次按钮弱一点
- 不要两个按钮抢视觉重点

---

## 6. 页面级 UI 升级方案

## 6.1 首页

文件：`app/(tabs)/index`

### 结论

首页整体不建议大改，仅做细节增强。

### 保留项

- 顶部问候区
- 统计条
- 快捷入口区
- 今日课程区
- 待办提醒区

### 可优化项

- 快捷入口图标尺寸和宫格节奏可再统一
- 今日课程卡和待办提醒卡的阴影强度可略统一
- 待办提醒的紧急状态可更明确

### 不建议

- 不建议重做首页结构
- 不建议把首页做得更复杂

---

## 6.2 我的

文件：`app/(tabs)/profile`

### 结论

“我的”页面视觉表现已经很好，建议保留整体结构。

### 可优化项

- 菜单卡之间的节奏可更统一
- 统计卡的图标、数字、标签的上下间距可再细调
- 退出登录按钮可以略弱化，不必过度抢眼

### 不建议

- 不建议大改头部信息卡

---

## 6.3 学生页

文件：`app/(tabs)/students`

### 当前问题

- 页面进入后直接是搜索栏和按钮，缺少“页面摘要”
- 学生总数和男女统计信息层级偏弱
- 搜索、导入、新增属于同一操作区，但视觉上有点散
- 班级切换区和统计区之间关系还不够清晰

### 建议结构

1. 页面摘要卡
2. 操作区
3. 班级切换
4. 学生列表

### 建议新增摘要卡内容

- 当前班级名称
- 学生总人数
- 男生人数
- 女生人数

### 摘要卡建议样式

- 使用白色卡片或主色浅底卡片
- 左侧放班级信息
- 右侧放统计数字
- 保持一眼能看懂“当前在看哪个班”

### 操作区建议

- 搜索栏为主
- 导入、新增为辅助操作
- 三者保持统一高度和对齐

### 列表卡建议

- 姓名最突出
- 性别标识减弱但保留
- 学号与家长信息作为次信息
- 卡片右箭头弱化

### 视觉关键词

- 清楚
- 干净
- 有秩序

---

## 6.4 学生详情页

文件：`app/student/[id]`

### 当前优点

- 模块已经比较丰富
- 成绩、考勤、家长信息都具备展示基础

### 优化建议

- 头像信息卡保持重点区风格
- 成绩概览与考勤统计的分区标题需要更统一
- 分析类小卡不要太多太碎，适当合并
- 编辑弹窗继续沿用统一弹窗按钮结构

### 注意

- 这页信息量大，优化重点不是更花，而是减少碎片感

---

## 6.5 成绩页

文件：`app/(tabs)/scores`

### 当前问题

- “考试列表”和“成绩分析”虽然已分 Tab，但页内视觉差异不够明显
- 顶部统计卡存在，但还不够像页面摘要区
- 考试卡片信息齐全，但状态层次还可以更强
- 分析页缺少当前分析口径提示

### 建议结构

1. 顶部摘要卡
2. Tab 切换
3. 条件口径提示
4. 列表或分析内容
5. FAB 区

### 摘要卡建议

摘要区建议强调：

- 本学期考试数
- 待录入数
- 已完成数

可以用更整齐的三栏统计结构，强化“这是成绩管理首页”的感觉。

### 考试卡片建议

- 保留左侧学科色条
- 右上角状态标签更明显
- 录入进度条更突出
- 已完成状态的数据区更规整

### 分析页建议

建议增加当前分析说明条，例如：

- 三年级2班
- 2025-2026 第二学期
- 当前考试：期中考试
- 当前科目：语文

### FAB 建议

- 主按钮：创建考试
- 次按钮：导入
- 两者层级更清楚

---

## 6.6 考试详情页

文件：`app/exam/[id]`

### 优化重点

- 顶部信息区更清楚展示考试名称、班级、科目、满分
- 成绩输入区强化“正在录入”的节奏感
- 统计小卡和列表分区层次更清楚

### 视觉建议

- 顶部信息区建议用浅底摘要卡包裹
- 输入框状态统一：已录入、未录入、缺考
- 保存按钮保持强主操作感

---

## 6.7 考勤页

文件：`app/attendance`

### 当前问题

- 记录页和统计页功能都有，但头部不够统一
- 日期、班级、批量操作之间略显分散
- 统计页新增了按学生视图后，未来要进一步强化“统计看板感”

### 建议结构

1. 页面状态卡
2. 一级 Tab
3. 日期与班级信息区
4. 操作区或统计切换区
5. 主列表 / 主统计区

### 页面状态卡建议内容

- 当前日期
- 当前班级
- 异常人数或当日出勤率

### 记录页建议

- 日期切换和“今天”按钮做成统一区块
- 班级切换和批量操作保持同一层级
- 学生卡的异常状态更醒目

### 统计页建议

- “按日查看 / 按学生查看”做成明确的二级切换器
- 概览卡和统计列表之间留出明确层次
- 学生统计卡中，出勤率最突出，迟到/请假/缺席作为次级指标

---

## 6.8 课程表页

文件：`app/(tabs)/schedule`

### 当前问题

- 页面功能最丰富，顶部操作区相对拥挤
- 班级、周次、编辑、导入、图例、筛选等都存在，容易显得堆
- 课程表本体已经不错，主要问题集中在页面头部组织

### 建议结构

1. 顶部信息区
2. 周次与今日定位
3. 操作区
4. 课程表主体
5. 图例区

### 顶部信息区建议

- 班级切换
- 当前学期信息
- 当前第几周

### 今日定位建议

- 做成轻量状态按钮
- 在当前周时显示“当前周”
- 不在当前周时显示“今日定位”或“回到本周”

### 操作区建议

- 编辑
- 导入
- 筛选

建议与班级/周次区在视觉上分层，避免全部挤在同一区域。

### 图例建议

- 图例本身保留
- 但需要控制密度与边距
- 强调“筛选用图例”，而不是让图例看起来像第二个工具栏

---

## 6.13 作业页

文件：`app/homework`

- 筛选、搜索、统计、列表之间层次明确
